Content Intelligence provides CLI access to entity data and aggregated intelligence from multiple sources. It enables AI tools, automation scripts, and developers to access comprehensive content data through Drush commands.

Why Content Intelligence?

Modern AI tools and automation workflows need structured access to Drupal content data. Content Intelligence bridges this gap by providing:

  • Entity-agnostic design — works with all content entity types (nodes, taxonomy terms, media, users, etc.)
  • Comprehensive data extraction — get entity fields, metadata, and intelligence from multiple plugins in a single request
  • CLI-first approach — optimized for AI tool consumption via Drush commands with JSON output
  • Plugin architecture — extensible system using PHP 8 Attributes for easy integration

Drush Commands

All commands support --format=json for machine-readable output.

Command Description
drush ci:types List available entity types
drush ci:bundles [type] List bundles for an entity type
drush ci:fields [type] [bundle] List fields for an entity type/bundle
drush ci:plugins List available intel plugins
drush ci:list [type] [bundle] List entities of a given type
drush ci:entity [type] [id] Get full intelligence data for an entity
drush ci:batch [type] Batch collect intel for multiple entities

Built-in Plugins

  • Statistics — Page view counts from the Statistics module (nodes only)
  • Content Translation — Translation status, coverage metrics, and missing languages

Contrib Integrations

  • Analyze module — Content analysis data from Analyze plugins (readability, SEO, etc.)
  • AI Social Posts — Social media posts linked to content

Creating Custom Plugins

Extend Content Intelligence with your own data sources using PHP 8 Attributes:

<?php

#[ContentIntel(
  id: 'my_plugin',
  label: new TranslatableMarkup('My Plugin'),
  description: new TranslatableMarkup('Custom intelligence data.'),
  entity_types: ['node'],
  weight: 50,
)]
class MyPlugin extends ContentIntelPluginBase {

  public function collect(ContentEntityInterface $entity): array {
    return [
      'metric' => 123,
      'status' => 'active',
    ];
  }

}

Use Cases

  • AI Content Analysis — Feed entity data to AI tools for content audits, recommendations, and improvements
  • Automation Scripts — Build workflows that process content based on field values and metadata
  • Content Reports — Generate comprehensive reports combining data from multiple modules
  • Migration Tools — Export structured content data for migrations or backups

Requirements

  • Drupal 10.3+ or 11.x
  • PHP 8.1+
  • Drush 12+

Optional Dependencies

  • Statistics — enables view count tracking
  • Content Translation — enables translation status
  • Analyze — enables content analysis plugins
  • AI Social Posts — enables social post tracking

Quick Start

  1. Install: composer require drupal/content_intel
  2. Enable: drush en content_intel
  3. Configure plugins at /admin/config/content/content-intel
  4. Test: drush ci:entity node 1 --format=json
